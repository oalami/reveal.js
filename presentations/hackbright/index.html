The why of Firebase


Let's look to software of the past...

Software of the past

70s: Mainframes (Thin Client)
80s: Personal Computer (Thick Client)
90s: Rise of the Web (Thin Client)

Today's mobile apps


  • Connected Thick Client
  • Mobile device does some of the work
  • Work offline (usually)
  • Usually request-response

Today's web apps

  • Powerful browsers
  • Mostly server side processing
  • Usually request-response

Tomorrow's web apps

  • New JavaScript frameworks are enabling more complex apps
  • Computing is moving closer to the user


What matters to users?

  • Fast apps
  • Stuff to update in realtime
  • Apps work offline

These exist today

  • Google Docs
  • Facebook Newsfeed
  • Live transit maps
